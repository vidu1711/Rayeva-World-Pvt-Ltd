# AI Systems Backend

Node.js + Express + MongoDB backend with two AI modules: **B2B Proposal Generator** and **Impact Reporting Generator**. Uses **OpenAI** or **Google Gemini** for generation (auto-detected from your API key) and stores results in MongoDB.

## Project structure

```
backend/
├── config/          # App and DB config
├── controllers/     # HTTP handlers
├── middleware/      # Request logging
├── models/          # Mongoose models (Proposal, ImpactReport, AILog)
├── prompts/         # AI prompt builders
├── routes/          # API routes
├── services/        # Business + AI logic
├── utils/           # Validators, JSON parsing
└── server.js
```

## Setup

1. **Install dependencies**

   ```bash
   cd backend
   npm install
   ```

2. **Environment**

   Copy `.env.example` to `.env` and set:

   - **AI (use one):** `OPENAI_API_KEY` (OpenAI `sk-...` or Google Gemini `AIzaSy...`) or `GEMINI_API_KEY` for Gemini
   - `MONGODB_URI` – default `mongodb://localhost:27017/ai-systems`
   - `PORT` – optional, default `3000`

   ```bash
   cp .env.example .env
   ```

3. **MongoDB**

   Have MongoDB running locally (or use a cloud URI in `.env`).

4. **Run**

   ```bash
   npm start
   ```

   Dev with auto-reload:

   ```bash
   npm run dev
   ```

   Server: `http://localhost:3000`

## API endpoints

### Health

- **GET** `/api/health` – Returns `{ status: "ok", timestamp }`

### Module 2: B2B Proposal Generator

- **POST** `/api/proposal/generate`

**Request body:**

```json
{
  "event_type": "Corporate conference",
  "budget": 2000,
  "sustainability_goal": "reduce plastic waste"
}
```

**Response (success):**

```json
{
  "success": true,
  "data": {
    "id": "<mongodb-id>",
    "proposal": {
      "suggested_product_mix": [
        { "product_name": "...", "quantity": "...", "estimated_cost": "..." }
      ],
      "budget_allocation": {},
      "estimated_cost_breakdown": {},
      "impact_summary": "..."
    }
  }
}
```

- Budget is enforced: proposal total must not exceed `budget`.
- Products are eco-friendly; output is valid JSON stored in DB.

---

### Module 3: Impact Reporting Generator

- **POST** `/api/impact/generate`

**Request body:**

```json
{
  "order_items": [
    {
      "product": "bamboo toothbrush",
      "quantity": 100,
      "material": "bamboo"
    }
  ]
}
```

**Response (success):**

```json
{
  "success": true,
  "data": {
    "id": "<mongodb-id>",
    "plastic_saved": "1000g",
    "carbon_avoided": "2500g",
    "local_sourcing_impact": "Products use bamboo materials.",
    "impact_statement": "Human-readable impact statement from AI."
  }
}
```

- **Logic:** `plastic_saved_grams = quantity × 10`, `carbon_avoided_grams = quantity × 25` (per item, summed).
- `impact_statement` is generated by the OpenAI API from these values.

## Example requests (curl)

**Proposal:**

```bash
curl -X POST http://localhost:3000/api/proposal/generate \
  -H "Content-Type: application/json" \
  -d "{\"event_type\":\"Corporate conference\",\"budget\":2000,\"sustainability_goal\":\"reduce plastic waste\"}"
```

**Impact:**

```bash
curl -X POST http://localhost:3000/api/impact/generate \
  -H "Content-Type: application/json" \
  -d "{\"order_items\":[{\"product\":\"bamboo toothbrush\",\"quantity\":100,\"material\":\"bamboo\"}]}"
```

## Features

- **AI:** OpenAI used for proposal JSON and impact statement; prompts in `prompts/`.
- **Logging:** All prompts and AI responses stored in `AILog`; request/response logging via middleware.
- **Validation:** Request bodies validated before processing; invalid JSON from AI is caught and returned as 400.
- **Error handling:** Try/catch in controllers; appropriate status codes (400, 503, 500).
- **Storage:** Proposals and impact reports saved to MongoDB (Proposal and ImpactReport models).

## Database models

- **Proposal:** `event_type`, `budget`, `sustainability_goal`, `generated_proposal`, `created_at`
- **ImpactReport:** `order_items`, `plastic_saved`, `carbon_avoided`, `impact_statement`, `created_at`
- **AILog:** `prompt`, `response`, `module_name`, `timestamp`
